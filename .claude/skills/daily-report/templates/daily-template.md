# Daily Report Template

Use this template to structure the daily report. Replace placeholders with actual content from the user's responses.

---

```markdown
# 日報 — <yyyy年mm月dd日>（<曜日>）

## 🌟 本日のハイライト

<一日で最も印象的だった出来事を1〜2文で。読者がこれだけ読んでも要点がわかるように。>

---

## 📋 実施事項

<取り組んだことを箇条書きで。具体的な成果・数値があれば添える。>

- **<カテゴリ/プロジェクト名>**: <何をしたか>
- **<カテゴリ/プロジェクト名>**: <何をしたか>
- **<カテゴリ/プロジェクト名>**: <何をしたか>

---

## 💡 気づき・学び

<新しく発見したこと、学んだことを記載。「なぜ重要か」を一言添える。>

1. **<気づきのタイトル>**
   <詳細な説明。背景や文脈を簡潔に。>

2. **<気づきのタイトル>**
   <詳細な説明。>

---

## 🚧 課題・ブロッカー

<現在直面している問題。対処の方向性があれば添える。>

| 課題 | 状況 | 対処の方向性 |
|------|------|-------------|
| <課題1> | <未着手 / 調査中 / 対応中> | <考えている解決策> |
| <課題2> | <未着手 / 調査中 / 対応中> | <考えている解決策> |

---

## 🎯 次の一手

<明日以降のアクション。優先度順に並べる。>

1. **[高]** <最優先のアクション>
2. **[中]** <次に取り組むこと>
3. **[低]** <時間があれば取り組むこと>

---

## 🔗 過去の日報との関連

<前回の「次の一手」の進捗状況や、繰り返し出現するテーマ、過去の気づきとの関連を記載。なければ省略可。>

- **前回からの進捗**: <前回予定していたことの進捗>
- **継続テーマ**: <繰り返し登場しているトピック>
- **関連する過去の気づき**: <過去の日報で記載した関連する学び>
```

## Writing Checklist

日報を書く際に確認すること：

- [ ] ハイライトだけ読んでも要点がわかるか
- [ ] 専門用語に補足があるか（読者は不特定多数）
- [ ] 具体的な数値・固有名詞が含まれているか
- [ ] 一文が長すぎないか（目安: 60文字以内）
- [ ] 「次の一手」に優先度がついているか
- [ ] 過去の日報との接続が自然か
